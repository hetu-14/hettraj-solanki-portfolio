How to use this guide: Work phase by phase. Don't jump ahead. Each phase builds on the last. Expect to spend 4–8 weeks per phase with 1–2 hours of daily practice. Phases 1–3 are your foundation. Phases 4–6 are your specialization and job-readiness layer.
Phase 01
Foundation — Linux, Networking & Git
Beginner
▶
Everything in DevOps runs on Linux. Before touching any tool, you must be comfortable at the command line, understand how networks work, and know how to track code with Git. This is non-negotiable.
🐧 Linux Basics
  • File system navigation
  • Permissions & users
  • Processes & services
  • Package management (apt/yum)
  • Shell scripting (Bash)
  • Cron jobs & scheduling
🌐 Networking
  • OSI / TCP-IP model
  • DNS, HTTP/S, SSH
  • IP addressing & subnets
  • Firewalls & iptables
  • Load balancing basics
📦 Git & Version Control
  • init, clone, commit, push
  • Branching strategies
  • Merge vs Rebase
  • Pull Requests & reviews
  • GitHub / GitLab workflows
🐍 Scripting
  • Bash scripting (primary)
  • Python basics (secondary)
  • Regex & text processing
  • Automation of sysadmin tasks

Container Orchestration — Kubernetes
Intermediate

📦 Core Objects
  • Pods & ReplicaSets

  • PersistentVolumes
🔐 Advanced K8s
  • Namespaces & RBAC
  • Ingress controllers
  • Helm charts
  • HPA & VPA (autoscaling)
  • StatefulSets & DaemonSets
🔄 GitOps
  • ArgoCD installation & setup
  • App of apps pattern
  • Sync policies & health
  • FluxCD alternative

Observability — Monitoring, Logging & SRE

  • Trace-log correlation
🛡 SRE Concepts
  • SLI, SLO, SLA definitions
  • Error budgets
  • Incident response runbooks

Certification Roadmap
Industry-recognized certs in the order most DevOps engineers should chase them. Each one builds on the last.
Strategy: Start with the AWS Cloud Practitioner as a foundation, then go for CKA (most in-demand hands-on cert), then the AWS DevOps Professional. Add Terraform Associate to complete the IaC story. For India market, AWS + CKA is the most valuable combination.
AWS Certified Cloud Practitioner
Amazon Web Services
Entry-level cloud literacy cert. Learn AWS services, billing, and architecture concepts. Not hands-on but builds vocabulary needed for everything else.
Entry Level
~$100 USD
90 min MCQ
3 year validity
Certified Kubernetes Administrator (CKA)
CNCF / Linux Foundation
Performance-based hands-on exam — you actually operate a real K8s cluster. Most in-demand cert for DevOps roles globally. 6,000+ job listings explicitly require it. Do this before AWS DevOps Pro.
Intermediate
$395 USD
2 hr hands-on
2 year validity
HashiCorp Certified: Terraform Associate
HashiCorp
Validates Infrastructure as Code skills with Terraform. 15,000+ US job postings mention it. Essential for cloud roles. Relatively easy to pass with 2 weeks of focused prep.
Intermediate
$70.50 USD
1 hr MCQ
2 year validity
AWS Certified DevOps Engineer – Professional
Amazon Web Services
Most in-demand cloud DevOps cert — 6,000+ job listings in the US alone. Covers CI/CD, automation, monitoring on AWS. Requires 2+ years AWS experience. This is your career milestone cert.
Advanced
$300 USD
3 hr scenario-based
3 year validity
Certified Kubernetes Application Developer (CKAD)
CNCF / Linux Foundation
Companion to CKA — focuses on application developers deploying to K8s. Stack with CKA for maximum credibility. Also hands-on performance-based exam.
Intermediate
$395 USD
2 hr hands-on
2 year validity
Certified Kubernetes Security Specialist (CKS)
CNCF / Linux Foundation
Requires CKA first. Covers RBAC, network policies, runtime security, supply chain security. Separates senior DevOps engineers from junior ones. Increasingly expected for mid/senior roles.
Advanced
$395 USD
2 hr hands-on
2 year validity
Google Cloud Professional DevOps Engineer
Google Cloud
Best-in-class SRE emphasis. Excellent second cloud cert. GCP's Kubernetes tooling (GKE) is industry-leading. Good add-on after AWS DevOps Pro.
Advanced
$200 USD
2 hr scenario-based
2 year validity
Real-World Projects
Build these in order. Each one produces a GitHub portfolio artifact you can show in interviews.
Rule: Use real cloud environments (AWS Free Tier), not just browser labs. Real cloud experience — IAM permissions, networking quirks, cost optimization — is what separates candidates. Every project should be pushed to GitHub with a README.
Beginner
Project 1: Dockerize a Web Application
Take any simple web app (Python Flask, Node.js, or even WordPress), write a Dockerfile, build the image, run it locally, then push to Docker Hub. Then write a docker-compose.yml to add a database service.
Docker
Docker Compose
Docker Hub
Git
✓ Portfolio win: Shows you can containerize any application — core DevOps skill #1
Beginner
Project 2: CI/CD Pipeline with GitHub Actions
Build a pipeline that: (1) runs tests on every push, (2) builds a Docker image on merge to main, (3) pushes the image to Docker Hub automatically. Include a badge in your README showing build status.
GitHub Actions
Docker
Docker Hub
✓ Portfolio win: Demonstrates CI/CD automation from code commit to deployable artifact
Intermediate
Project 3: Deploy to Kubernetes on AWS (EKS)
Deploy your Dockerized app to an EKS cluster. Set up Deployments, Services, and an Ingress controller (Nginx). Implement HPA so it scales automatically under load. Use Helm for packaging.
Kubernetes

Project 4: Infrastructure with Terraform
